# Authoritative Source Reference

## Purpose

Structured reference for Phase 2.5 (Best Practices and Authoritative Source Research). Contains five layers of authoritative open-access resources organized by domain relevance.

## How SPARK Uses This Document

1. During Phase 2.5, SPARK identifies which layers are relevant based on the product
2. For each relevant institution, SPARK searches for published frameworks and best practices
3. Findings are presented with adoption recommendations (Adopt / Adapt / File / Reject)
4. The user decides what enters the product definition -- SPARK never auto-incorporates

## Search Quality Rules

- Do NOT search entities scoring below 5.5 on the scorecard (references/scorecard.md)
- Regulatory authority sources ALWAYS override consulting firm opinions when they conflict
- For multi-jurisdiction products, run regulatory sweeps independently per jurisdiction
- If a product touches payments in any form, ALWAYS search payment-specific regulators

## Search Strategy

Query patterns:
- Consulting firms: "[firm name] [product topic] framework [year]"
- Regulatory bodies: "[authority] [topic] requirements guidance"
- Behavioral/actuarial: "[firm name] [topic] best practices research"

Assess each finding on:
1. **Applicability** -- does it apply to this product/user/market?
2. **Recency** -- published within last 2-3 years?
3. **Actionability** -- can we translate to specific product decisions?

Only present findings scoring well on all three dimensions.
