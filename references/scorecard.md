# SparkKit Entity Scorecard — Reference File
## For use by spark-phase-2.5-bestpractice and spark-compliance skills

### How to Use This Reference

When conducting authoritative source sweeps in Phase 2.5 or compliance research:

1. **Look up the relevant domain** for the research topic (e.g., "Legal Advice" for IP questions, "Regulatory Compliance" for compliance frameworks).
2. **Prioritize by tier:** Start with ★★★ entities (score 8.5+), then ★★ (7.0-8.4), then ★ (5.5-6.9).
3. **Do NOT search entities scored below 5.5** (marked with —). Their open-access content is too limited to be useful.
4. **Use the URL column** as the starting point for web searches within that entity's knowledge center.
5. **Note the region code** to prioritize jurisdiction-relevant sources for PassportCard's markets (IL, DE, EU, UK, US, AU).
6. **Check the Niche Sub-Domains column** to match the entity's strength to the specific research question.

### Scoring Criteria

| Criterion | Description |
|-----------|-------------|
| Content (1-10) | Amount and depth of published research, frameworks, and data |
| Access (1-10) | Ease of finding and accessing content. Open-access vs. gated |
| Recency (1-10) | How current content is and publishing cadence |
| Action (1-10) | How directly frameworks translate into product decisions for PassportCard |

### Tier System

| Tier | Score Range | Usage |
|------|-------------|-------|
| ★★★ | 8.5-10.0 | Primary source for Phase 2.5 sweeps. Always search first. |
| ★★ | 7.0-8.4 | Strong supplementary source. Search when ★★★ sources are insufficient. |
| ★ | 5.5-6.9 | Niche value. Search only when topic matches their specific sub-domain. |
| — | below 5.5 | Do not search. Consider direct engagement only if specifically needed. |
